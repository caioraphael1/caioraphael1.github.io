PollInputEvents

  • How it works :

    • As I understand and based on tests:

      • (2025-05-28) Windows x64:

        • The OS does not know the concept of "pressed" and "released". That is built by RayLib/GLFW.

          • A simple comparison is done: "if the previous state was not DOWN in my array and now the OS reports it as DOWN, then it will be marked as PRESSED in my array".

        • I tested for both Key inputs and Mouse inputs; the explanation applies to both.

    • Ex :

      • If the poll is done every 5 seconds and I press a key at 2.5s, then when polling occurs it will be detected that the key was JUST PRESSED, even though 2.5s have already passed.

        • The same applies to released.

App

  • Game Loop .

    • Great introductory video.

    • .

    • Not event based.

      • Something is always drawn. There is always something running.

    • Draw

      • Only when EndDrawing  is called is everything submitted to the GPU for drawing.

      • Input is also updated after EndDrawing , ready for the next frame.

    • Input:

      • It is constant during the whole frame.

      • Calling IsKeyDown()  multiple times in the same frame will always return the same result.

    • FrameRate.

      • By default, FPS is unlimited.

        • This makes the CPU run at 100%.

      • DrawFPS()  shows the FPS.

        • It actually draws the FPS on the screen.

        • Very useful.

      • SetTargetFPS()  caps the FPS.

        • Not a hard guarantee, but an upper limit, meaning FPS can be lower than that.

    • Showing and Hiding:

      • To hide something on screen, just don't draw it.

      • It's your job to determine what is drawn each frame.

    • Animation:

      • Use the delta between frames, since frame rate is inconsistent.

  • Pause the game .

    • Simply use state variables.

    • Do not use sleep, that will make the game look crashed to Windows.

  • DrawTextEx with LoadFont / LoadFontEx (TrueType ‚Üí Bitmap Atlas) :

    • Loads a TTF font into a texture atlas at a specified size and renders via DrawTextEx() .

    • Advantages

      • Supports any TTF/OTF font and Unicode ranges you include.

      • Can specify base font size to optimize crispness at that size.

      • Kerning and spacing correctly handled.

    • Disadvantages

      • Atlas size is fixed; scaling beyond the loaded size leads to aliasing.

      • Larger atlases consume more VRAM.

      • You must choose target font sizes in advance.

    • "Fix" blur :

      • Important :

        • Use MULTIPLES of the font's 'base font size'.

          • To do this, open the font on fontdrop.info  and check the unitsPerEM .

            • Interpret that afterwards.

          • Often this does not fully fix the issue.

      • Note :

        • Doing rl.SetTextureFilter(text_font.texture, .POINT)  did not change anything.

      • Scale the font correctly .

  • Signed Distance Field (SDF) Fonts with Custom Shader :

    • SDF example in RayLib .

    • Pre‚Äêgenerate an SDF font atlas (e.g. using msdfgen) and render with a shader that interprets the distance field.

    • Advantages

      • Exceptionally sharp at arbitrary scales and rotations.

      • Easy to apply outlines, glows, and other effects in the shader.

      • Single atlas can cover multiple font sizes.

    • Disadvantages

      • Requires integrating a custom GLSL shader.

      • Generation of SDF atlas is an extra build step.

      • Slight learning curve to tune smoothing parameters.

  • Multi‚ÄêSize Bitmap Atlases (e.g. BMFont / AngelCode) :

    • Generate several bitmap atlases at different sizes (e.g. 16, 32, 64 pt) with an external tool and load the closest match at runtime.

    • Advantages

      • Each atlas is optimized for its specific size‚Äîvery crisp.

      • No custom shaders needed.

      • Fine control over kerning and glyph packing via the tool.

    • Disadvantages

      • Multiple atlases increase memory usage.

      • Requires runtime logic to select and switch atlases.

      • Doesn‚Äôt scale smoothly between discrete sizes.

  • Render to High-Resolution RenderTexture and Downsample :

    • Draw text at a larger RenderTexture resolution and then scale down to screen size.

    • Advantages

      • Can improve edge smoothness via hardware filtering.

      • No external tools or shaders required.

    • Disadvantages

      • Extra GPU/memory overhead for the larger RenderTexture.

      • May introduce slight blurriness if downsample filter isn‚Äôt ideal.

      • More complex render logic and framebuffers.

Timers

  • Create Timer .

    • Builds the concept of "Timer" using a Struct and functions outside that struct, i.e., without OOP; cool.

    • .

    • Lifetime here symbolizes remaining duration.

    • It updates the timer and then checks if it finished.

  • Create timer .

    • Does not update the Timer, only checks if it finished.

    • Lifetime here symbolizes total duration. 

    typedef struct Timer {
    double startTime;   // Start time (seconds)
    double lifeTime;    // Lifetime (seconds)
} Timer;

void StartTimer(Timer *timer, double lifetime)
{
    timer->startTime = GetTime();
    timer->lifeTime = lifetime;
}

bool TimerDone(Timer timer)
{
    return GetTime() - timer.startTime >= timer.lifeTime;
}

double GetElapsed(Timer timer)
{
    return GetTime() - timer.startTime;
}
