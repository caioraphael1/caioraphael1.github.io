Backends from Godot

Render

  • Vulkan:

    • Primary renderer in the Godot 4.x series.

    • Used for high-end platforms (Windows, Linux, macOS with MoltenVK, Android).

    • Full PBR (Physically Based Rendering) pipeline.

    • GPU-driven mesh culling and rendering.

  • OpenGL (Compatibility):

    • Legacy renderer from Godot 3.x, kept for compatibility and low-end devices.

    • Used as a fallback or for platforms without Vulkan support.

    • Labeled as "Compatibility renderer".

Audio

  • Fully internal, written in C++.

  • Modular: uses an AudioDriver abstraction layer.

    • Backends:

      • PulseAudio, ALSA (Linux)

      • CoreAudio (macOS/iOS)

      • WASAPI (Windows)

      • OpenSL ES or AAudio (Android)

  • Features:

    • Real-time mixing

    • Buses and effects system

    • Support for 3D audio (via AudioStreamPlayer3D )

SDL

  • Uses Input singleton backed by platform-specific event capture.

  • SDL-like platform abstraction in platform/  modules.

  • Supports:

    • Keyboard, mouse, touch, gamepads.

    • Android sensor input (accelerometer, gyroscope).

    • IME support for complex text input.

  • Gamepad input via SDL GameControllerDB-compatible mapping layer.

Physics

  • Two built-in backends:

    • GodotPhysics: Default as of 4.x (internal implementation).

    • Jolt Physics: Optional backend (can be enabled at compile time).

    • Bullet Physics is no longer used in 4.x.

Text

  • TextServer abstraction:

    • TextServerFallback (default, software-based)

    • TextServerAdvanced (uses HarfBuzz  + FreeType  + shaping for advanced layout)

  • Supports complex text layout, RTL, ligatures.
